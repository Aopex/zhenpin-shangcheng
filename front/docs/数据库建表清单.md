# 数据库建表清单

> 本项目是一个 **微信小程序电商商城**，包含首页、分类、购物车、订单、个人中心、商品详情六大模块。以下共设计 **12 张表**。

---

## 1. `user` — 用户表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 用户主键 |
| `openid` | VARCHAR(64) | UNIQUE, NOT NULL | 微信 openid，小程序登录凭证 |
| `union_id` | VARCHAR(64) | NULL | 微信 unionid，跨平台标识（可选） |
| `nickname` | VARCHAR(64) | DEFAULT '微信用户' | 用户昵称 |
| `avatar_url` | VARCHAR(512) | NULL | 头像地址 |
| `phone` | VARCHAR(20) | NULL | 手机号（需用户授权） |
| `gender` | TINYINT | DEFAULT 0 | 性别：0-未知 1-男 2-女 |
| `status` | TINYINT | DEFAULT 1 | 状态：0-禁用 1-正常 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 最后更新时间 |

> **字段说明：**
> - `openid`：`app.js` 中 `wx.login()` 获取 code 后换取的用户唯一标识，是小程序体系中最核心的用户标识。
> - `union_id`：如果同一主体有公众号、App 等多端，可通过 union_id 打通用户体系。
> - `nickname`/`avatar_url`：「我的」页面顶部展示用户头像和昵称（当前代码为 mock 固定图片和"点击登录"文案）。
> - `phone`：配合微信手机号快速登录组件使用，后续可扩展。
> - `status`：用于管理账号封禁等运营场景。

---

## 2. `category` — 商品分类表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 分类主键 |
| `name` | VARCHAR(32) | NOT NULL | 分类名称 |
| `icon_url` | VARCHAR(512) | NULL | 分类图标地址（SVG/PNG） |
| `sort_order` | INT | DEFAULT 0 | 排序权重，数值越大越靠前 |
| `parent_id` | BIGINT | DEFAULT 0 | 父分类 ID，0 表示顶级 |
| `status` | TINYINT | DEFAULT 1 | 状态：0-隐藏 1-显示 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

> **字段说明：**
> - `name`：首页「金刚区」和分类页「左侧边栏」直接展示的分类名称，如"数码家电"、"服饰鞋包"等（代码中 mock 了 8~10 个分类）。
> - `icon_url`：首页金刚区每个分类下方的图标（对应 `/assets/icons/icon_rhgp8pamq9n/` 下的 SVG 文件）。
> - `sort_order`：控制分类在金刚区和侧边栏中的显示顺序。
> - `parent_id`：预留二级分类能力，当前代码只用了一级分类，后续可扩展子分类。
> - `status`：运营可控制某些分类是否对用户可见。

---

## 3. `product` — 商品表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 商品主键 |
| `product_no` | VARCHAR(32) | UNIQUE, NOT NULL | 商品编号（如 P101、D101） |
| `title` | VARCHAR(256) | NOT NULL | 商品标题 |
| `price` | DECIMAL(10,2) | NOT NULL | 售价 |
| `original_price` | DECIMAL(10,2) | NULL | 原价（划线价） |
| `category_id` | BIGINT | FK → category.id | 所属分类 |
| `main_image` | VARCHAR(512) | NOT NULL | 商品主图 URL |
| `stock` | INT | DEFAULT 0 | 总库存 |
| `sales` | INT | DEFAULT 0 | 累计销量 |
| `status` | TINYINT | DEFAULT 1 | 状态：0-下架 1-上架 |
| `sort_order` | INT | DEFAULT 0 | 排序权重 |
| `description` | TEXT | NULL | 商品描述（富文本） |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

> **字段说明：**
> - `product_no`：代码中作为商品跳转参数传递（`toDetail` 通过 `data-no` 传入 `product-detail` 页面），是商品的业务编号。
> - `title`：商品标题，首页卡片、分类列表、购物车、订单中均有展示，代码中限制两行省略。
> - `price`：核心展示字段，首页/分类/详情/购物车/订单均展示，使用红色主题色 + `Bahnschrift` 字体突出显示。
> - `original_price`：商品详情页展示的划线价（`¥{{productDetail.originalPrice}}` 带删除线样式）。
> - `stock`/`sales`：商品详情页底部显示"销量 1258 / 库存 500"，SKU 弹窗也展示库存。
> - `main_image`：列表场景下的缩略图（首页瀑布流、分类列表、购物车列表）。
> - `status`：软删除+上下架控制，下架商品不展示。

---

## 4. `product_image` — 商品图片表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `product_id` | BIGINT | FK → product.id | 关联商品 |
| `image_url` | VARCHAR(512) | NOT NULL | 图片地址 |
| `image_type` | TINYINT | DEFAULT 1 | 类型：1-轮播主图 2-详情长图 |
| `sort_order` | INT | DEFAULT 0 | 排序权重 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

> **字段说明：**
> - `image_type = 1`（轮播主图）：对应商品详情页顶部的 `banners` 数组，代码中展示为全屏 swiper 轮播，支持点击大图预览 `wx.previewImage`。
> - `image_type = 2`（详情长图）：对应详情页下方的 `detailImages` 数组，纵向排列的商品介绍图文。
> - 与 `product` 表拆分是因为一个商品有多张图片（轮播 3~4 张 + 详情 2~3 张），一对多关系，避免主表字段冗余。

---

## 5. `product_sku` — 商品 SKU 表（规格）

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | SKU 主键 |
| `product_id` | BIGINT | FK → product.id | 关联商品 |
| `sku_no` | VARCHAR(32) | UNIQUE, NOT NULL | SKU 编号 |
| `spec_values` | VARCHAR(256) | NOT NULL | 规格值组合（如 "朱砂红,M"） |
| `price` | DECIMAL(10,2) | NULL | SKU 级别定价（为空则取商品价） |
| `stock` | INT | DEFAULT 0 | 该规格库存 |
| `image_url` | VARCHAR(512) | NULL | 规格对应图片 |
| `status` | TINYINT | DEFAULT 1 | 状态：0-禁用 1-启用 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

> **字段说明：**
> - `spec_values`：商品详情页 SKU 弹窗中展示的"颜色"和"尺码"组合。代码中 mock 了颜色（朱砂红/柿子橙/象牙白）和尺码（S/M/L），部分规格标记为 `disabled`（如 L 码无库存）。
> - `price`：支持 SKU 级别独立定价（如不同颜色不同价格），为空时回退到商品表价格。
> - `stock`：对应代码中 SKU 弹窗的"库存 N 件"展示，以及规格 tag 的 `disabled` 状态控制（库存为 0 则不可选）。
> - `image_url`：选择不同规格时可切换商品主图，提升购物体验。

---

## 6. `spec` — 规格名表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `product_id` | BIGINT | FK → product.id | 关联商品 |
| `name` | VARCHAR(32) | NOT NULL | 规格名（如"颜色"、"尺码"） |
| `sort_order` | INT | DEFAULT 0 | 排序 |

> **字段说明：**
> - 代码中 SKU 弹窗有两组规格：`颜色`（sku-group）和 `尺码`（sku-group），每组下有多个可选 tag。
> - 该表定义规格维度名称，配合 `spec_value` 表使用，使规格体系可灵活扩展（如增加"材质"、"套餐"等）。

---

## 7. `spec_value` — 规格值表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `spec_id` | BIGINT | FK → spec.id | 关联规格名 |
| `value` | VARCHAR(32) | NOT NULL | 规格值（如"朱砂红"、"M"） |
| `sort_order` | INT | DEFAULT 0 | 排序 |

> **字段说明：**
> - 存储每个规格名下的可选值。例如 `spec_id` 对应"颜色"时，此表存储"朱砂红"、"柿子橙"、"象牙白"三个值。
> - 代码中 SKU 弹窗的 `.sku-tag` 元素即对应此表的记录，选中状态（`.active`）和禁用状态（`.disabled`）由库存决定。

## 8. `banner` — 轮播图表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `title` | VARCHAR(64) | NULL | 轮播图标题（管理用） |
| `image_url` | VARCHAR(512) | NOT NULL | 图片地址 |
| `link_type` | TINYINT | DEFAULT 1 | 跳转类型：1-商品详情 2-分类页 3-小程序页面 4-外部链接 |
| `link_value` | VARCHAR(256) | NULL | 跳转目标（商品编号/分类ID/页面路径/URL） |
| `sort_order` | INT | DEFAULT 0 | 排序权重 |
| `status` | TINYINT | DEFAULT 1 | 状态：0-隐藏 1-显示 |
| `start_time` | DATETIME | NULL | 生效开始时间 |
| `end_time` | DATETIME | NULL | 生效结束时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

> **字段说明：**
> - `image_url`：对应首页顶部 `banner-swiper` 中的图片，代码中使用 Unsplash 图片，宽高比 750×300。
> - `link_type` + `link_value`：控制点击轮播图后的跳转行为，最常见是跳转到某个商品详情（传 `product_no`）。
> - `start_time` / `end_time`：运营可配置轮播图的展示时间段（如活动期间展示、活动结束自动下线）。
> - `sort_order`：控制轮播图的展示顺序。

## 9. `cart_item` — 购物车表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `user_id` | BIGINT | FK → user.id | 所属用户 |
| `product_id` | BIGINT | FK → product.id | 关联商品 |
| `sku_id` | BIGINT | FK → product_sku.id | 关联 SKU |
| `quantity` | INT | DEFAULT 1 | 购买数量 |
| `checked` | TINYINT | DEFAULT 1 | 是否选中：0-未选 1-选中 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 加入时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

> **字段说明：**
> - `user_id`：每个用户独立的购物车，代码中当前为 mock 数据，真实场景需按用户隔离。
> - `product_id` + `sku_id`：购物车中每个条目对应一个具体的 SKU（如"朱砂红 + M"），代码中 `cartItems` 的每个对象有 `no`（商品编号）和具体的商品信息。
> - `quantity`：对应购物车中 stepper 控件的 `count` 字段，支持 + / - 操作（代码中限制最少 1 件）。
> - `checked`：对应购物车中每个商品前的复选框状态，影响结算栏的"合计"计算和"全选"状态。

## 10. `order` — 订单表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 订单主键 |
| `order_no` | VARCHAR(32) | UNIQUE, NOT NULL | 订单编号（如 DD20231001001） |
| `user_id` | BIGINT | FK → user.id | 下单用户 |
| `status` | VARCHAR(16) | NOT NULL | 订单状态 |
| `total_count` | INT | NOT NULL | 商品总件数 |
| `total_amount` | DECIMAL(10,2) | NOT NULL | 实付金额 |
| `freight_amount` | DECIMAL(10,2) | DEFAULT 0 | 运费 |
| `discount_amount` | DECIMAL(10,2) | DEFAULT 0 | 优惠金额 |
| `address_snapshot` | JSON | NULL | 下单时地址快照 |
| `remark` | VARCHAR(256) | NULL | 买家备注 |
| `is_deleted` | TINYINT | DEFAULT 0 | 用户侧逻辑删除标记：0-正常 1-已删除 |
| `pay_time` | DATETIME | NULL | 支付时间 |
| `ship_time` | DATETIME | NULL | 发货时间 |
| `receive_time` | DATETIME | NULL | 收货时间 |
| `finish_time` | DATETIME | NULL | 完成时间 |
| `cancel_time` | DATETIME | NULL | 取消时间 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 下单时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**状态枚举值：**

| 值 | 含义 | 对应代码 Tab |
|---|---|---|
| `UNPAID` | 待付款 | `待付款` |
| `UNSHIPPED` | 待发货 | `待发货` |
| `UNRECEIVED` | 待收货 | `待收货` |
| `FINISHED` | 已完成 | `已完成` |
| `CANCELLED` | 已取消 | （取消操作后） |

> **字段说明：**
> - `order_no`：订单编号，对应订单列表页的"订单号：DD20231001001"展示。
> - `status`：订单页顶部 Tab 筛选的核心字段，代码中定义了 5 个 Tab（全部/待付款/待发货/待收货/已完成），`filterOrders()` 方法按此字段过滤。不同状态对应不同的操作按钮：
>   - `UNPAID` → 「取消订单」+「去支付」
>   - `UNSHIPPED` → 「提醒发货」（前端本地记录次数，第 3 次提醒后模拟商家发货，状态进入 `UNRECEIVED`）
>   - `UNRECEIVED` → 「确认收货」
>   - `FINISHED` → 「删除订单」
> - `total_count` / `total_amount`：对应订单卡片底部"共N件商品 实付金额：¥xxx"。
> - `address_snapshot`：下单时将收货地址 JSON 快照存入订单，即使用户后续修改地址也不影响历史订单。
> - `is_deleted`：用于用户侧逻辑删除订单，删除后订单列表、订单详情和个人中心统计默认不再展示。
> - 各时间字段：记录订单生命周期的关键节点，便于查询和数据统计。

## 11. `order_item` — 订单商品表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `order_id` | BIGINT | FK → order.id | 关联订单 |
| `product_id` | BIGINT | FK → product.id | 关联商品 |
| `sku_id` | BIGINT | FK → product_sku.id | 关联 SKU |
| `product_no` | VARCHAR(32) | NOT NULL | 商品编号（冗余快照） |
| `title` | VARCHAR(256) | NOT NULL | 商品标题（下单时快照） |
| `image_url` | VARCHAR(512) | NOT NULL | 商品图片（下单时快照） |
| `spec_text` | VARCHAR(128) | NULL | 规格描述（如"朱砂红,M"） |
| `price` | DECIMAL(10,2) | NOT NULL | 下单时单价 |
| `quantity` | INT | NOT NULL | 购买数量 |
| `subtotal` | DECIMAL(10,2) | NOT NULL | 小计金额 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

> **字段说明：**
> - `title` / `image_url` / `price`：下单时的商品信息快照，即使商品后续被修改或删除，订单中仍保留原始信息。对应订单列表中每个商品的标题、图片和价格展示。
> - `spec_text`：记录下单时选择的规格（如"朱砂红, M, 1件"），对应商品详情页 SKU 选择后的 `selectedSkuText`。
> - `quantity`：对应订单商品列表中的"x{{item.count}}"。
> - `subtotal`：`price × quantity`，用于对账和退款计算。
> - 单订单多商品：代码中订单列表支持多件商品横向滑动展示（`scroll-view` + `multi-goods`），因此需要 order_item 表存储多条记录。

## 12. `address` — 收货地址表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `user_id` | BIGINT | FK → user.id | 所属用户 |
| `name` | VARCHAR(32) | NOT NULL | 收货人姓名 |
| `phone` | VARCHAR(20) | NOT NULL | 联系电话 |
| `province` | VARCHAR(32) | NOT NULL | 省 |
| `city` | VARCHAR(32) | NOT NULL | 市 |
| `district` | VARCHAR(32) | NOT NULL | 区/县 |
| `detail` | VARCHAR(256) | NOT NULL | 详细地址 |
| `is_default` | TINYINT | DEFAULT 0 | 是否默认地址：0-否 1-是 |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

> **字段说明：**
> - 「我的」页面有「地址管理」菜单入口（`menu-item`），用户可在此维护收货地址列表。
> - `is_default`：标记默认地址，下单时自动填充。同一用户仅允许一条默认地址。
> - `province` + `city` + `district` + `detail`：拆分存储便于后期按地区统计分析，也方便使用微信地址选择组件。
> - 下单时将地址信息 JSON 快照存入 `order.address_snapshot`，保证历史订单数据完整性。

## 表关系总览

```
user (1) ──── (N) cart_item
user (1) ──── (N) order
user (1) ──── (N) address

category (1) ──── (N) product

product (1) ──── (N) product_image
product (1) ──── (N) product_sku
product (1) ──── (N) spec
spec    (1) ──── (N) spec_value

order (1) ──── (N) order_item

product_sku (1) ──── (N) cart_item
product_sku (1) ──── (N) order_item
```

## 页面-表映射关系

| 页面 | 使用的数据表 | 说明 |
|---|---|---|
| **首页** | `banner`, `category`, `product` | 轮播图、金刚区分类、商品瀑布流 |
| **分类页** | `category`, `product`, `product_image` | 左侧分类导航、右侧分类商品列表 |
| **商品详情** | `product`, `product_image`, `product_sku`, `spec`, `spec_value` | 主图轮播、价格信息、SKU 弹窗选择 |
| **购物车** | `cart_item`, `product`, `product_sku` | 购物车列表、数量修改、选中状态、结算 |
| **订单页** | `order`, `order_item` | 订单列表、状态 Tab 筛选、操作按钮 |
| **个人中心** | `user`, `order`, `address` | 用户信息、订单快捷入口、地址管理 |
